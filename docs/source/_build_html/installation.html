Installation on Ubuntu 16.04¶

1. julia programs needed¶

get:

Pkg.add("RobotOS")
Pkg.add("PyCall")

It would also be wise to:

Pkg.build("RobotOS")
Pkg.build("PyCall")

2. Gazebo¶

To install:

curl -ssL http://get.gazebosim.org | sh

Then to run:

gazebo

More information is here

3. ROS¶

To get it:

sudo apt-get install python-rospkg

sudo apt-get ros-kinetic-gazebo-ros-pkgs
sudo apt-get ros-kinetic-simulators

http://wiki.ros.org/ROS/EnvironmentVariables

http://wiki.ros.org/kinetic/Installation/Ubuntu

3. Initialization of Package¶

3.1. Run cmake . and make¶

  1. open another terminal -> ctr``+``shift``+``t
  2. navigate to the folder .scr/c_code
  3. in the terminal type cmake . and hit enter
  4. in the terminal type make and hit enter

3.2. Set path for gazebo plugins¶

3.3 Get the velodyne_hdl32 model for Gazebo¶

Either:

  1. Fork the gazebo_models database by visiting https://bitbucket.org/osrf/gazebo_models/fork.

OR

  1. Just copy the folder ./LiDAR/scr/gazebo/gazebo_models into $HOME.gazebo/models/

Note

press ctr + h to show hidden folders (..like .gazebo)

Potential Issues¶

1. running using RobotOS in julia -> fails¶

This may be due to path issues was fixed here.

  • Additional info here
  1. I had some issues with ros

http://wiki.ros.org/UpstreamPackages

so I removed it:

sudo aptitude remove python-rospkg
sudo apt-get remove ros-*

sudo apt-get purge ros-*
sudo apt-get autoremove

sudo apt-get purge '^ros-*'

If you do not have aptitude:

sudo apt install aptitude

Then I did:

sudo aptitude -f install ros-kinetic-desktop-full
sudo apt install python-rosdep
rosdep update
rosdep check --from-paths . --ignore-src --rosdistro kinetic
rosdep install --from-paths . --ignore-src --rosdistro kinetic -y
